# Skills Manager 웹 리빌드 계획

> `프론트리팩토링` 워크플로우 **1단계(리팩토링 계획)** 산출물.
> 대상: `skills-manager-web/`(원본 `skills-manager` 레포의 복제본 — 원본 불변).

## 문제 / 목표

Skills Manager는 지금 **터미널(CLI) 도구**라, 비개발자 일반인에게는 "검은 화면에 명령어 타이핑"이 진입장벽이다. (코드 자체는 이미 모듈별로 깔끔 — 10파일·1,208줄.)

목표: **비개발자가 클릭으로 쓰는 웹 화면**을 입힌다.
① 빠른 로딩 ② 기능별 파일 정리 ③ 헷갈리지 않는 구조 ④ 비개발자가 직접 쓰고 유지보수.

## 환경 결정 — 빌드도구 0 (Vite 안 씀)

- 작은 로컬 도우미 서버 `serve.js`(Node, **의존성 0**)가 기존 `scan.js`를 돌려 결과(JSON)만 웹에 넘긴다.
- 브라우저는 보안상 `~/.claude` 폴더를 직접 못 읽으므로 이 도우미가 다리 역할.
- `127.0.0.1`(이 PC)에만 열려 외부 노출 없음. **읽기 전용** 원칙 유지(검사만, 변경 없음).
- 비개발자는 `node serve.js` 한 번이면 브라우저가 열린다(나중에 더블클릭 아이콘으로 감쌀 수 있음).

## 파일 구조 (종류별 분리)

```
skills-manager-web/
  serve.js          # 로컬 도우미(브라우저 ↔ scan.js 다리)
  scan.js 등        # 기존 검사 로직 — 그대로 재사용(다시 안 짬)
  web/
    index.html      # 화면 뼈대 + 본문 글
    css/style.css   # 디자인(색·간격) — 모양 고치는 곳
    js/app.js       # 화면 그리기·이동
  baseline/         # 2단계 기준선(검사 결과 스냅샷)
```

## 화면

- **홈**: 큰 버튼 4개 = 지금의 4모드(겹치는 스킬 찾기 · 이 작업 뭐 쓰지 · 내 워크플로우 · 스킬 정리).
- **① 겹침 지도**: 터미널 막대그래프를 진짜 웹 막대로. *(첫 슬라이스에서 구현)*
- ②③④: 다음 단계에서 차례로.

## 워크플로우 단계 매핑

| 단계 | 적용 | 상태 |
|---|---|---|
| 1. 계획 | 이 문서 | ✅ |
| 2. 브라우저 동작 테스트 | 첫 화면이 브라우저에서 뜨는지 + 검사 숫자가 기준선과 같은지 | 진행 |
| 3. 코드 단순화 | 웹 파일 군더더기 없이 | 대기 |
| 4. 시각 점검 | design-review | 대기 |
| 5. 성능 점검 | 로딩 속도 측정·개선("로딩 길다" 해소) | 대기 |
| 6. 코드 리뷰 | 최종 점검 | 대기 |

> ※ 이 워크플로우는 원래 "있는 프론트엔드 고치기"용이지만, Skills Manager엔 웹이 없어 1·3단계는 "새로 짓기"로 결을 맞춰 적용한다. 4·5·6단계는 새 빌드에도 그대로 맞는다.

## 범위 밖
- 검사 로직 재작성(기존 scan.js 재사용) · settings·다른 스킬 변경(읽기 전용 유지) · Vite/번들러 도입.

## 메모
- 원본 `skills-manager` 레포는 건드리지 않는다(이 폴더는 복제본·실험장).
- 이 웹 버전이 쓸 만해지면 원본에 합칠지는 나중에 결정.
